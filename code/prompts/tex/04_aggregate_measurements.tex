\begin{tcolorbox}[breakable,enhanced,colback=blue!3,colframe=blue!40!black,title={Prompt},fonttitle=\bfseries,left=8pt,right=8pt,top=6pt,bottom=6pt]

Deduplicate measurement instruments (benchmarks, datasets, user studies, surveys, human evaluations, and other named evaluation procedures) by grouping rows that refer to the \textbf{same instrument} under different names. Do NOT split or merge based on reported metrics or scores.

Each input row has an \texttt{id}. Output only the grouping as \texttt{\{canonical\_name: [ids]\}}. All metadata is resolved from the IDs programmatically --- do NOT repeat descriptions, papers, or snippets.

\textbf{Canonical name rules:} community-standard short name of the instrument. Name the \textbf{instrument only} (e.g. "MMLU", "HELM Math", "User preference study"), not "MMLU Accuracy" or metric-laden labels. Include subset qualifier when needed. For human evaluations, use a descriptive label that captures the method.

\textbf{Grouping rules:}
\begin{itemize}
\item Group the same instrument referred to differently (e.g. "MMLU" and "Massive Multitask Language Understanding"; "User study" and "Human preference evaluation" when they clearly describe the same procedure)
\item DO NOT group different subsets (e.g. "HELM Math" vs "HELM Code", "MMLU" vs "MMLU Math")
\item DO NOT group distribution-shift variants as the parent (e.g. "ImageNet-A" $\neq$ "ImageNet")
\item DO NOT merge dataset variants with different experimental conditions
\item DO NOT merge distinct facets or modalities of an instrument (e.g. "HELM Math" and "HELM Code" are separate facets, not synonyms)
\item DO NOT merge different types of human evaluation (e.g. "User preference study" $\neq$ "Expert annotation evaluation")
\item When in doubt, keep separate
\end{itemize}

\textbf{Output format:}
\begin{verbatim}
{"groups": {"MMLU": [0, 4], "HumanEval": [2], "User preference study": [6, 9]}, "ungrouped": [1, 3, 5]}
\end{verbatim}
\texttt{ungrouped} lists IDs that don't match any other row. Every input ID must appear exactly once.
