# Abductive reasoning
**Type:** Construct  
**Referenced in:** 12 papers  
**Also known as:** Abductive code reasoning  

## State of the Field

Across the provided literature, abductive reasoning is most commonly defined as the ability to find the most likely explanation or hypothesis for a set of incomplete observations. This framing appears in multiple contexts, with one paper noting it "starts with incomplete observations and seeks the most likely explanation" (OlympicArena) and another describing it as "generating a possible explanation from incomplete information" (GameArena). A slightly different definition characterizes it as a latent ability to infer solutions using prior knowledge and case-specific analysis to simplify problems (SATER). A more specialized variant, "abductive code reasoning," is defined as finding the simplest explanation (the input) for a given program and its output by abstractly understanding the program's purpose. The construct is frequently studied as a type of logical reasoning, alongside deductive, inductive, and analogical reasoning. In terms of operationalization, abductive code reasoning is measured using the CruxEval and LiveCodeBench benchmarks, while in another context, the game of Taboo is used to evaluate a model's abductive capabilities.
