# Domain-specific reasoning
**Type:** Construct  
**Referenced in:** 5 papers  

## State of the Field

Across the provided sources, domain-specific reasoning is framed in two distinct contexts, with no consensus on a single definition. One paper defines it as the latent ability to apply knowledge within a specific technical domain, such as Linux systems, to interpret user utterances and diagnose issues. This work suggests that models struggle with "subtler and more context-dependent instances" that require this form of reasoning. In contrast, another paper presents an alternative definition, casting it as the ability to apply psychological theories and expert-like reasoning in emotional support contexts. This framing is associated with theories like the "Helping Skills Theory" and involves analyzing emotional states and user intentions. While both definitions center on applying specialized knowledge, they point to different application areas—one technical and problem-oriented, the other interpersonal and therapeutic. The provided data does not specify how this construct is measured or operationalized in either context.
