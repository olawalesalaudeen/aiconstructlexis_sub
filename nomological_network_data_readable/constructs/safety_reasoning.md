# Safety reasoning
**Type:** Construct  
**Referenced in:** 4 papers  

## State of the Field

Across the provided literature, safety reasoning is characterized as a latent, internal process by which a model makes safety-related decisions. This process involves determining if a user request is harmful and then formulating an appropriate and safe course of action, which can include refusing the request. While the sources agree on its nature as a latent capability, they offer different levels of specificity. One paper defines it broadly as a 'latent cognitive process' for identifying harm. In contrast, another work provides a more detailed account, describing it as the ability to 'analyze both explicit and implicit intents' across multimodal inputs, reconstruct the user's request, and assess risks. This latter framing explicitly connects the construct to 'chain-of-thought-based safety decision-making capabilities.' The literature also touches on how this capability is acquired, with one paper noting that alignment approaches often presume models learn it implicitly, while another proposes a framework to explicitly train models to 'internalize' this reasoning ability.
