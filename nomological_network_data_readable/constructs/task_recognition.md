# Task recognition
**Type:** Construct  
**Referenced in:** 5 papers  
**Also known as:** Task differentiation  

## State of the Field

Task recognition is most commonly defined as a language model's ability to identify a target task from in-context demonstrations and then solve it by leveraging its pre-existing knowledge from pre-training. This prevailing view, articulated in "Investigating the Pre-Training Dynamics of In-Context Learning," posits that the model recognizes the task pattern from examples rather than learning the task anew. A related framing, under the name "Task differentiation," describes the ability to distinguish between tasks based on their input characteristics without explicit labels, as seen in the context of enhancing a task router's capabilities. Across the provided sources, the construct is consistently situated within the process of In-Context Learning (ICL). It is also frequently studied in contrast to "Task learning," with some papers aiming to disentangle these two phenomena. The provided data conceptualizes Task recognition as a latent ability but does not specify any measurement instruments or behaviors used to operationalize it.
