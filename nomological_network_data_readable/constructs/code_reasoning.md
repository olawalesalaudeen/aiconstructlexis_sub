# Code reasoning
**Type:** Construct  
**Referenced in:** 8 papers  

## State of the Field

Across the surveyed literature, the most common framing of code reasoning is as a latent ability to reason about program execution and code behavior. Some papers offer more specific definitions, describing it as the ability to analyze and infer conclusions about program logic, or more operationally, to predict a code snippet's output from an input or infer the input from a known output. This construct is consistently operationalized and measured using specific benchmarks. CruxEval is frequently cited as an instrument for its evaluation, with papers also mentioning DS-1000 and CodeMMLU, which assesses the ability through tasks like "code repair, execution reasoning, and fill-in-the-blank challenges." These tasks are described as assessing how a model can "reason about the execution of simple Python programs." Code reasoning is also positioned as related to logical reasoning, with one source stating it "encompasses a category of tasks that demonstrate logical reasoning through code." A single paper proposes a directional relationship where planning is a cause of code reasoning.
