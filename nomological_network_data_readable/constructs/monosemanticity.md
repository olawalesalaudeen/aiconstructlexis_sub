# Monosemanticity
**Type:** Construct  
**Referenced in:** 10 papers  

## State of the Field

Across the provided literature, monosemanticity is characterized as a property of learned features extracted from model activations, commonly pursued to achieve interpretability. The concept is defined in two related ways: one framing describes it as the degree to which a feature corresponds to a "single word meaning rather than conflating multiple meanings in context," while another offers a broader definition of a feature corresponding to a "single coherent concept rather than multiple unrelated behaviors." The more specific, word-level definition is elaborated upon in one study, which operationalizes the analysis of this construct by "focusing on polysemous words." This work suggests a goal of being able to "extract monosemantic features from polysemantic activations." Sparse Autoencoders (SAEs) are presented as one method for this purpose, described as "a promising solution for extracting monosemantic and interpretable features." The overall aim is to acquire features that are individually understandable while preserving the semantic relationships between concepts.
