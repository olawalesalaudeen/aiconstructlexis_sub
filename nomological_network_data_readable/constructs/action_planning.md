# Action planning
**Type:** Construct  
**Referenced in:** 4 papers  

## State of the Field

Action planning is characterized in at least two distinct ways across the provided literature, depending on the application domain. One framing defines it as the ability to generate "executable sequences of low-level keyboard and mouse actions" for general computer control, translating high-level skills into specific operations based on goals and context ("Cradle: Empowering Foundation Agents towards General Computer Control"). A different line of work operationalizes it within software engineering as the process of generating a sequence of "read, write, or modify actions" to guide tasks like test generation ("Otter: Generating Tests from Issues to Validate SWE Patches"). In this latter context, the planner might start by executing read actions on relevant files and functions. The construct is also studied alongside Self-reflection, where a model is instructed to "self-reflect on the proposed plan," suggesting a process where planning is followed by an evaluative step.
