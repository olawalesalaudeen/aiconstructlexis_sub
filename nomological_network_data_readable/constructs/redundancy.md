# Redundancy
**Type:** Construct  
**Referenced in:** 3 papers  

## State of the Field

Based on the provided literature, redundancy is defined as the distribution of features across multiple neurons rather than their concentration in a single unit. This characteristic is consistently framed as a mechanism for supporting model robustness against perturbations such as noise or dropout. The concept is studied in relation to polysemanticity, with one paper noting that "polysemantic code learned by these networks favors redundancy." This is further contrasted with monosemantic code, which redundancy is said to discourage. To operationalize this construct, the available data indicates that redundancy is measured using LLM-based evaluation.
