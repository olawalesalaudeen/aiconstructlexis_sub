**Type:** Measurement  
**Referenced in:** 6 papers  

## State of the Field

Reflexion is consistently described across the provided sources as a self-improvement approach for code generation. The prevailing characterization is that of an iterative procedure that uses execution results or feedback to refine generated code, with one definition specifying this refinement occurs "through reasoning." In some contexts, it is framed more specifically as an "iterative debugging procedure" and is used as a baseline for evaluating code refinement techniques. The available data situates Reflexion alongside other similar methods, such as Self-Editing and Self-Debugging, that focus on code improvement.
