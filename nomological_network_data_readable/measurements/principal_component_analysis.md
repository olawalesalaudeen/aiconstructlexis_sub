# Principal component analysis
**Type:** Measurement  
**Referenced in:** 8 papers  
**Also known as:** PCA, PCA Visualization, Principal Component Analysis  

## State of the Field

Across the provided literature, Principal Component Analysis (PCA) is consistently described as a dimensionality-reduction technique, though its specific applications vary. One use is to analyze a model's internal states by applying PCA to "residual stream activations" to visualize the "geometric structure of numerical representations across model layers" (RethinkMCTS: Refining Erroneous Thoughts inMonteCarlo Tree Search for Code Generation). Another application treats PCA as a "lens function" for projecting embeddings into a low-dimensional space, a process which "emphasizes directions of high variance" and highlights "global variance" (Does Acceleration Cause Hidden Instability in Vision Language Models? Uncovering Instance-Level Divergence Through a Large-Scale Empirical Study). A third documented use is for analyzing behavioral data, where it serves to "cluster co-occurring follow-up types into prototypical collaboration behaviors" (TORSO: Template-Oriented Reasoning Towards General Tasks). In all these cases, PCA is operationalized as an analytical tool to make high-dimensional data—whether from model internals, embeddings, or interaction logs—more interpretable by revealing underlying patterns.
