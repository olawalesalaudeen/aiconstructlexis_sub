# StableToolBench
**Type:** Measurement  
**Referenced in:** 6 papers  

## State of the Field

Across the provided literature, StableToolBench is consistently described as a benchmark for evaluating language models, with a focus on their ability to use tools. The most common framing is that it measures the "accuracy of function calls generated by models" ("Benchmarking Agentic Workflow Generation"), while other papers describe its purpose more broadly as evaluating "LLM tool invocation" ("Graphically Speaking: Unmasking Abuse in Social Media with Conversation Insights") or "tool use in LLM agents" ("MoRAgent: Parameter Efficient Agent Tuning with Mixture-of-Roles"). In practice, papers use StableToolBench to operationalize and measure the constructs of "Tool use" and "Generalization". Multiple sources note that the benchmark is derived from ToolBench and contains 765 solvable tasks or questions distributed across six subsets. These subsets are characterized as varying in "tool category and instruction complexity, ranging from single-tool to multi-tool instructions" ("Graphically Speaking: Unmasking Abuse in Social Media with Conversation Insights"). A frequently mentioned feature is its stability, which is attributed to a "stable API caching mechanism" or a "virtual API server" and "stable evaluation system". Researchers employ it in evaluations alongside other benchmarks, such as RestBench-TMDB, BFCL, GSM8K, and MATH, to compare the performance of different models.
