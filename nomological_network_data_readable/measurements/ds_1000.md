# DS-1000
**Type:** Measurement  
**Referenced in:** 16 papers  

## State of the Field

Across the provided literature, DS-1000 is consistently characterized as a benchmark used to evaluate code generation within data science scenarios. It is defined as containing 1000 problems, described as "data science workflows" or "code completion problems," that cover a range of widely-used Python libraries including NumPy and Pandas. One source specifies that the benchmark "introduces non-standalone functions," highlighting problems with contextual dependencies. The evaluation mechanism involves assessing generated code against test cases, and some papers note that it supports both "completion and insertion" modes. The most prevalent application of DS-1000, as supported by evidence from multiple papers, is to measure the behavior of `Code generation`. In addition to code generation, one paper states it is used to evaluate "code reasoning and libraries usage capabilities." The benchmark is also listed as a measurement instrument for `Code editing`, `Faithfulness`, `Text generation`, and `Code efficiency`, although the provided data does not include specific evidence detailing these uses.
