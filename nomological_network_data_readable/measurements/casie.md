# CASIE
**Type:** Measurement  
**Referenced in:** 4 papers  

## State of the Field

Across the provided literature, CASIE is consistently characterized as a dataset for event extraction and event detection within the cybersecurity domain. The papers agree on its core purpose, which is to identify security-related incidents from sources described as technical reports, cyber threat reports, and news. Several sources specify that CASIE defines five event types: Databreach, Phishing, Ransom, Discover, and Patch. One common definition further notes that the dataset includes "trigger and argument annotations." In practice, CASIE is used in experimental evaluations, sometimes alongside other information extraction benchmarks, to assess model performance. For instance, it is explicitly used as one of several datasets to measure the `Generalization` of a proposed method.
