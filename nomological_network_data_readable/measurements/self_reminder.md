# Self-Reminder
**Type:** Measurement  
**Referenced in:** 3 papers  

## State of the Field

Based on the provided data, Self-Reminder is characterized as a training-free defense method. Its operational mechanism, as described in "RED: Unleashing Token-Level Rewards from Holistic Feedback via Reward Redistribution", involves adding safety reminders both before and after a user's request. This process is applied during the inference stage, meaning it does not require any model retraining. The stated purpose of this technique is to act as a defense, as one source notes, "Self-Reminder defends by adding safety reminders before and after user requests."
