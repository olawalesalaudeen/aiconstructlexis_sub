# Token copying
**Type:** Behavior  
**Referenced in:** 3 papers  

## State of the Field

As defined in the provided literature, token copying is the task of identifying and reproducing a specific token from a given position, such as the first or last, within an input prompt. This behavior is also described more broadly as the "‘copying’ or ‘recalling’ of a single or multiple tokens from the prompt." The source material frames it as a "seemingly simple" task that can be challenging for Transformer-based LLMs, resulting in errors. It is studied alongside the behavior of counting. According to one paper, model collapse is a phenomenon that can cause performance degradation on this task.
