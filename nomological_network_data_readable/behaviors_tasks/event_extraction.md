# Event extraction
**Type:** Behavior  
**Referenced in:** 3 papers  

## State of the Field

Across the surveyed literature, event extraction is consistently described as a task of obtaining structured information from unstructured text. The specific scope of this behavior, however, is framed in two distinct ways. One definition presents it as identifying and structuring "specific life events" with properties like date and people from "conversational text." In this context, it is used as an LLM-driven function for processing memories from conversations. A second, more general definition describes the task as identifying and classifying "event triggers and their associated arguments" from general unstructured text. This framing characterizes it as a "challenging task in natural language processing" that obtains "poly-element structured information regarding what happens." The performance of this behavior is operationalized and evaluated using "sentence-level and document-level EE benchmark datasets."
