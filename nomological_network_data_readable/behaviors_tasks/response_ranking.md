# Response ranking
**Type:** Behavior  
**Referenced in:** 7 papers  
**Also known as:** Pairwise ranking, Solution reranking  

## State of the Field

Across the provided literature, response ranking is most commonly defined as the task of ordering multiple candidate responses from best to worst based on an assessment of their quality. This behavior is operationalized by using a 'judge model' or 'ranker', which can be an LLM, to perform the ordering. The assessment can be based on various criteria, such as a 'constructed knowledge base' or assigned scores. While the general definition is prevalent, some papers focus on more specific applications. For instance, 'pairwise ranking' is described as the narrower task of determining the relative order of just two items. Another specific application is 'solution reranking', which involves reordering generated reasoning paths using a scoring mechanism, such as an 'aggregation score from a PRM', to select the most accurate final answer. The provided data also shows that response ranking is studied in relation to response aggregation, with one paper noting that using 'rankers before fusers' is a particularly effective configuration.
