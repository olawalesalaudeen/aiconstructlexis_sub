# AI-generated text detection
**Type:** Behavior  
**Referenced in:** 13 papers  
**Also known as:** LLM-generated text detection, Machine-generated text detection, LLM text identification, Self-written text recognition, Model identification, Machine text detection, Copyrighted book detection  

## State of the Field

Across the provided literature, AI-generated text detection is most commonly defined as the task of classifying a given text as either human-written or machine-generated. This is frequently described as a "pressing need" to distinguish between the two sources ("BiScope: AI-generated Text Detection by Checking Memorization of Preceding Tokens"). While the dominant framing involves classifying an entire text, a more granular version of this behavior, termed `LLM text identification`, seeks to identify specific LLM-written segments within a mixed-origin document. A smaller set of studies investigates related but distinct tasks, such as `Self-written text recognition`, where a model discriminates its own writing, and `Copyrighted book detection`, a specific application for identifying excerpts from pretraining data. Another related task, `Model identification`, focuses on determining model lineage rather than the origin of a specific text. To operationalize this behavior, researchers employ various methods, which are broadly categorized as "training-based and training-free methods." Specific approaches mentioned for measurement include using Min-K% Prob, leveraging "conditional probability curvature" via tools like Fast-DetectGPT, or applying a "CREATIVITY INDEX" as a criterion for detection.
