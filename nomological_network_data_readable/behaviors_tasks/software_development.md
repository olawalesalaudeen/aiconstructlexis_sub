# Software development
**Type:** Behavior  
**Referenced in:** 3 papers  
**Also known as:** ML library development, Automated AI development  

## State of the Field

Across the surveyed literature, software development is framed in several distinct ways, ranging from general application creation to specialized AI system engineering. One paper defines it broadly as the production of runnable software artifacts like games or web apps that pass functional checks, noting that this can be approached with multi-agent systems composed of roles like a 'Requirement Designer,' 'Code Developer,' and 'Tester' ("MetaAgent: Automatically Constructing Multi-Agent Systems Based on Finite State Machines"). A second, more specialized framing is 'ML library development,' which is the process of creating machine learning operators for a specific hardware architecture and programming language ("Adaptive Self-improvement LLM Agentic System for ML Library Development"). This variant is described as a challenging task requiring expertise in both ML algorithms and the target language. A third perspective treats it as 'automated AI development,' an end-to-end process of building a complete AI system from a user query, encompassing data processing, modeling, and reporting. This behavior is studied using resources like the DevAI dataset, which contains real-world tasks for this purpose ("Agent-as-a-Judge: Evaluate Agents with Agents").
