# Code navigation
**Type:** Behavior  
**Referenced in:** 3 papers  
**Also known as:** Code repository navigation  

## State of the Field

Across the provided literature, `Code navigation` is characterized in two complementary ways. One definition frames it as the task of understanding and navigating a code repository to answer questions or perform actions, while another describes it as the process of an agent exploring a codebase by moving between files and code entities. The behavior is operationalized using benchmarks such as `LongBench`, which is cited as being used for code repository navigation tasks. One paper suggests a technical approach to facilitate this behavior, using a "graph representation" of the repository for "structured navigation." `Code navigation` is reported to be enabled by `Complex reasoning`, with one study noting that it can "take advantage of multi-step reasoning." The concept is also presented as being related to `Code understanding`.
