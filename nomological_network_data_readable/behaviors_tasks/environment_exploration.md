# Environment exploration
**Type:** Behavior  
**Referenced in:** 4 papers  

## State of the Field

Across the provided literature, environment exploration is consistently defined as an agent's observable action of interacting with an environment to gather information and update its internal state representation. The operationalization of this behavior varies by domain. One line of work situates it in physical environments, where an agent explores to "verify whether the conditions (e.g., temperature and TV status) are met" (Exploratory Retrieval-Augmented Planning For Continual Embodied Instruction Following). Another application frames it within a software engineering context, where agents "autonomously explore and inspect the codebase environment by executing various commands" to understand the system state, a process facilitated by tools like OpenHands. In both settings, the stated purpose is information gathering to build a representation of the environment, such as establishing "environmental context memory" or developing a "comprehensive understanding of the codebase for out-of-sync recovery" (SyncMind: Measuring Agent Out-of-Sync Recovery in Collaborative Software Engineering). This behavior is also studied in relation to task planning.
